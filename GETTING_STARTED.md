# GETTING_STARTED.md — はじめに読む手引き（BoulderAILab）

このファイルは、Claude Code を使って **BoulderAILab** を作り始めるための、初学者向けの手順書です。上から順にやれば進めます。分からなくなったら、その時点の内容をそのまま Claude Code に貼り付けて「これで合ってる？」と聞いて大丈夫です。

GitHubリポジトリ：https://github.com/kenta-yos/boulder-ai-lab

---

## 0. まず全体像：ファイルは3つあります

- `CLAUDE.md`：Claude Code が毎回自動で読む「プロジェクトの説明書」。あなたが触る必要はほぼありません。
- `SPEC.md`：詳しい仕様の参照用。`CLAUDE.md` から参照されます。
- `GETTING_STARTED.md`：このファイル。あなた（人間）向けの手順書。

この3つを1つのフォルダに入れて、そのフォルダで Claude Code を動かします。

---

## 1. 事前準備：料金について（先に読んでください）

Claude Code を使うには、Claude の有料プランか、API（Console）アカウントが要ります。無料のClaude.aiプランでは使えません。

あなたはすでに Console（API）アカウントを持っていますが、支出上限が月$5に設定されています。アプリを作る作業は、1回30〜60分のセッションで $0.50〜$3 ほどかかるので、$5だとすぐ上限に達します。おすすめは次のどちらか。

- 案A：Console の「制限を調整」から月の上限を少し上げる（例：$20〜$30）。使った分だけの従量課金。
- 案B：Claude Pro（月$20）などのサブスクに入り、Claude Code をそのアカウントで使う。定額なので使いすぎの心配が少なく、初学者にはこちらが安心。

どちらでも進められます。迷ったら案Bが分かりやすいです。

## 2. Claude Code をインストールする

初学者には、ターミナルを使わない「デスクトップアプリ」が一番かんたんです。

### 方法A：デスクトップアプリ（おすすめ・ターミナル不要）
1. ブラウザで `https://claude.com/download` を開く。
2. お使いのOS（Mac / Windows）版をダウンロードしてインストール。
3. アプリを開き、Claudeアカウントでログイン。

### 方法B：ターミナルで入れる（少し慣れている人向け）
ターミナル（Macなら「ターミナル」アプリ、WindowsならPowerShell）を開いて、下のコマンドを貼り付けて実行します。

- Mac / Linux の場合：
  ```bash
  curl -fsSL https://claude.ai/install.sh | bash
  ```
- Windows（PowerShell）の場合：
  ```powershell
  irm https://claude.ai/install.ps1 | iex
  ```

入ったか確認するには、ターミナルで次を実行してバージョン番号が出ればOKです。
```bash
claude --version
```
うまくいかないときは、次を実行すると原因を診断してくれます。
```bash
claude doctor
```

## 3. リポジトリを手元に落として、3ファイルを置く

GitHubのリポジトリは作成済みなので、それを手元のパソコンに落とす（clone する）ところから始めます。

1. ターミナルで、分かりやすい場所に移動してから次を実行する（例では書類フォルダ）。
   ```bash
   cd ~/Documents
   git clone https://github.com/kenta-yos/boulder-ai-lab.git
   ```
   これで `boulder-ai-lab` フォルダが手元にできます。
2. その `boulder-ai-lab` フォルダの中に、`CLAUDE.md` / `SPEC.md` / `GETTING_STARTED.md` の3ファイルを入れる。

（clone のときに英語のメッセージが出ても、エラー表示がなければ成功です。不安なら画面をそのまま Claude Code に貼って確認してもらってOK。）

---

## 4. Claude Code を起動する

### デスクトップアプリの場合
アプリを開き、作業フォルダとして手順3で clone した `boulder-ai-lab` フォルダを開く（または選ぶ）。

### ターミナルの場合
作ったフォルダに移動してから `claude` を起動します。`cd` の後ろは、実際のフォルダの場所に置き換えてください。
```bash
cd ~/Documents/boulder-ai-lab
claude
```
初回は認証のためブラウザが開きます。案内に従ってログインしてください。

---

## 5. 最初にやること（作り始める前の準備）

Claude Code が起動したら、まず下の文章をそのまま貼り付けて送ってください。

> まず CLAUDE.md と SPEC.md を読んで、このプロジェクトの全体像を日本語でやさしく要約してください。私はコーディング初学者です。そのうえで、Phase 0 で作るものだけに絞って、作業の計画を立てて見せてください。まだファイルは作らず、計画だけ先に見せてください。なお、このプロジェクトは GitHub リポジトリ（https://github.com/kenta-yos/boulder-ai-lab）とつながっています。私は git 操作に不慣れなので、区切りのいいところで「変更を保存して GitHub に上げる（コミットとプッシュ）」のを、コマンドを省略せず教えながら、あなたが主導してやってください。

このとき覚えておくと安心な操作：

- Claude Code はファイルを変更する前に必ず確認を出します。変更内容（差分）が表示され、「はい（適用）」「はい、以降このセッションでは確認しない」「いいえ（理由を伝える）」から選べます。よく分からないうちは、1つずつ「はい」で進め、変なところだけ「いいえ」で止めればOKです。
- キーボードで Shift + Tab を押すと、モードが切り替わります。「Plan（計画だけ立てて実行しない）」モードにしておくと、いきなり作られる心配がなく安全です。

## 6. Phase 0 を作る

計画に納得できたら、次を貼り付けて実行を始めてもらいます。

> 計画でOKです。Phase 0 から始めましょう。まずは Next.js プロジェクトの土台を作り、動くところまで持っていってください。私が途中で何をすればいいか（コマンドやクリック操作）は、そのつど省略せず具体的に教えてください。1ステップ進むごとに、動作確認の方法も教えてください。

Phase 0 で目指すゴール（これができたら Phase 0 は完了）：

- パソコンのローカルでアプリが起動し、ブラウザで開ける。
- 短いボルダリング動画を選ぶと、数十秒以内に「敗因」と「処方」がテキストで表示される。
- その内容についてチャットで質問できる。
- 解析結果が記録一覧に残る。

途中で必要になること（Claude Code が案内してくれますが、心構えとして）：

- アプリがClaudeを呼ぶためのAPIキーを、`.env.local` というファイルに書く場面があります。このキーは絶対に他人に見せない・ネットに上げないでください（Claude Code もそう扱うよう指示済みです）。
- 「動画をどう扱うか（Blobストレージ）」や「データベース（Neon）」の接続が必要になったら、Claude Code が登録手順を案内します。分からなければ「この手順をもっと細かく教えて」と返してください。

## 7. Phase 1 以降の進め方

Phase 0 が動いたら、同じ要領で次に進みます。各フェーズの中身は `SPEC.md` の「段階的ロードマップ」にあります。次のように伝えるだけでOKです。

> Phase 0 が動作確認できました。次に SPEC.md の Phase 1 に進みたいです。まず計画を立てて見せてください。私は初学者なので、専門用語は避けて、手順は具体的にお願いします。

大事なルール：**1つのフェーズが動作確認できてから、次のフェーズに進む。** 一気に全部作ろうとしないこと。

## 8. 困ったときの小ワザ

- 何か動かない・設定が怪しいとき：ターミナルで `claude doctor` を実行すると診断してくれます。
- 会話が長くなって混乱してきたら：Claude Code の中で `/clear` と入力すると会話がリセットされます（`CLAUDE.md` の内容は毎回読み直されるので消えません）。
- 使うモデルを変えたいとき：`/model` と入力すると選べます。難しい設計は強いモデル、簡単な作業は軽いモデル、と使い分けると節約になります。
- 迷ったら、状況をそのまま日本語で書いて Claude Code に送ってください。「今こういう状態で、こう出た。どうすればいい？」で十分伝わります。

## 補足：このアプリは大きめのプロジェクトです

完成形（動画の数値解析やGPUを使う画像認識まで）はかなり本格的です。でも Phase 0 は、Claude API とブラウザだけで動く小さな範囲に絞ってあります。まずは Phase 0 を「自分の動画を上げたら敗因と処方が返ってくる」ところまで動かすのを、最初のゴールにしてください。そこまで来れば、あとは同じやり方の繰り返しです。
